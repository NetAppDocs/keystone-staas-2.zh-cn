= Keystone支持的存储容量
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
NetApp Keystone STaaS 服务支持多种类型的存储容量。了解这些不同的容量术语可以帮助您使用Keystone。



[[logical-capacity]]
== 逻辑容量

这是客户放置在Keystone基础设施上的数据。所有Keystone容量均指逻辑容量。例如，如果Keystone基础架构上存储了 1 TiB 的文件，则至少应购买 1 TiB 的容量。



[[committed-capacity]]
== (按已提交容量划分)

订阅期间每月计费的最小逻辑容量：

* 容量致力于每个性能服务级别。
* 承诺容量和额外的服务级别可以在期限内添加。




[[changes-to-committed-capacity]]
=== 承诺产能的变化

在订阅期限内，您可以更改承诺的容量。但是，有一定的前提条件：

* 根据特定条件，承诺容量可能会减少。有关信息，请参阅link:../concepts/capacity-requirements.html["产能削减"]。
* 承诺容量不能在您的订阅到期前 90 天增加，除非订阅期限额外续订 12 个月。
* 您可以通过控制台或向您的Keystone Success Manager (KSM) 请求更改已承诺的容量。有关申请更改的信息，请参阅 link:../concepts/support-services.html["NetApp Keystone支持"]。




[[consumed-capacity]]
== 已消耗容量

消耗容量是指当前在服务上消耗的容量（以存储的 TiB 为单位）。根据数据存储类型进行不同的计算：

* *统一或块优化存储*：消耗容量根据订购过程中选择的容量类型（逻辑或物理）计算。系统将为每个性能服务级别实例执行计算。
+
.. *逻辑容量*：它是以下各项的总和：
+
*** 计量逻辑容量，在提高存储阵列数据效率之前，用于存储所有实例和类型的客户数据，例如副本、镜像副本、版本和克隆。
*** 用于存储快照和某些克隆的元数据和差异数据的物理容量。
*** 任何厚配置的物理容量。


.. *物理容量*：它是以下各项的总和：
+
*** 在存储阵列数据效率之后，计量物理容量以存储所有实例和类型的客户数据，例如副本、镜像副本、版本和克隆。
*** 用于存储快照的元数据和差异数据的物理容量。
*** 任何厚配置的物理容量。




* *对象存储*：消耗的容量计算为用于在所有节点上存储所有实例和类型的客户数据的计量物理容量。此计算基于配置的信息生命周期管理 (ILM) 策略。
* *Cloud Volumes ONTAP*：消耗的容量计算为所有 Cloud Volumes ONTAP 卷的计量配置逻辑容量。




[[burst-capacity]]
== 突发容量

NetApp Keystone服务使您能够在服务级别承诺容量的基础上使用额外容量。这被称为突发容量使用。

请注意以下几点：

* 突发容量是在Keystone协议中约定的。它通常设置为高于承诺容量的 20%，并按照与承诺容量相同的费率收费。
* 突发容量可以弹性消耗，并按日平均消耗量收费。




[[billed-capacity]]
== 计费容量

月度费用 = (承诺容量 [TiB] * 承诺费率 [$/TiB]) + (每日平均预置突发容量 [TiB] * 突发费率 [$/TiB])。每月账单包含基于承诺容量的最低费用。

根据每日平均突发容量消耗，每月账单会超出最低费用。
